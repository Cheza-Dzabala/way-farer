Way Farer

This is a server side application for the Way Farer bus ticketing application.

Sign In [/auth/signin]

Sign In To Existing Account [POST]

Users can sign into their accounts using this action.
The response sends back a user object along with an authentication bearer token that is used for further authentication of the user.

  • Request (application/json)

     + Attributes
 
  + email (required)
  + password (required)

  • Response 200 (application/json)

    • Attributes

      • email
      • first_name
      • last_name
      • is_admin (boolean)
      • token

Sign Up [/auth/signup]

Create a New Account [POST]

User can create way farer accounts using this action.
The response sends back a new user object along with an authentication bearer token that is used for further authentication of the user.

  • Request (application/json)

    + Attributes
  
  + email (required)
  + first_name (required)
  + last_name (required)
  + password (required)

  • Response 201 (application/json)

    • Attributes

      • status
      • data (object)

    • Schema

        {
  "type": "object",
      "status": {
        "type": "string"
      },
      "data": {
          "type": "object",
          "user_id": "integer",
          "first_name": "string",
          "last_name": "string",
          "email": "string",
        }
  }

Create a trip [POST]

Amdins can create a trip using this action.

  • Request (application/json)

    • Headers

        token: bearer token-string

  • Response 201 (application/json)

    • Attributes
      • fare (number, required) - float
      • origin (required)
      • destination (required)
      • trip_date (required) - date
      • seating_capacity (number, required)

Trips - Patch [/trips/{id}/cancel]

Cancel a trip [PATCH]

Admins can cancel a trip by sending in a trip id as a URL parameter.
This action also requires that an authorization token be present in
the header.

  • Parameters

    • id (number, required) - The ID of the trip the user wants to cancel.

  • Request (application/json)

    • Headers

        token: bearer token-string

  • Response 200 (application/json)

    • Body

        status: 'success',
  data: {
      message: 'Trip cancelled successfully',
   },
